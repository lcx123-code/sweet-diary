# Sweet Diary — 整体设计方案 v3

> 综合全部规格：UI 设计系统 + 产品方向 + 设计思路 + AI 路线图

---

## 一、产品定位

**一句话：**
> 像写备忘录一样轻松，像翻纪念册一样温暖。

**不是：**
- 情侣打卡
- 聊天
- 社区
- 效率工具

**而是：**
- 两个人共同写的一本日记
- 记录只有彼此看得懂的日常碎片
- 核心场景：随手记录，长期回顾

---

## 二、设计系统

### 2.1 设计关键词

- **纸感**：像纸张一样柔和
- **留白**：大量空白
- **温暖**：米色基调，陶土点缀
- **克制**：只保留核心功能
- **呼吸感**：大行高 1.7x
- **编辑部风格**：杂志排版

### 2.2 禁止清单

渐变 / 玻璃拟态 / 高饱和粉色 / 复杂插画 / 厚重卡片 / 聊天气泡 / 大头像

### 2.3 颜色

- 背景 `#F8F6F2`
- 主文字 `#2E2E2E`
- 辅助文字 `#8A8A8A`
- 强调色 `#C98B7A`（陶土）
- 分割线 `#EAE5DE`
- 白色 `#FFFFFF`

### 2.4 字体

- 标题/日期：Noto Serif SC（思源宋体）
- 正文：系统字体（苹方 / HarmonyOS Sans）
- 不用手写体

### 2.5 字号

日期 40px / 标题 24px / 正文 18px / 辅助 14px

### 2.6 间距

只用 8/16/24/32

### 2.7 圆角

统一 16px

## 四、功能路线图

### Phase 1 — MVP
邮箱注册 → 6 位邀请码绑定 → 共同日记 → 块编辑器 → 图片上传 → 时间轴

### Phase 2 — 增长
AI 生成日记 / 照片故事 → 标签提取 → 纪念日看板

### Phase 3 — 差异化
AI 爱情时间轴 → 年度回忆 → 精美导出

### Phase 4 — 壁垒
AI 关系报告 → AI 回忆搜索 → 纪念册

---

## 五、AI 路线图

1. 照片/碎碎念 → 文字
2. 照片 → 故事
3. 内容 → 标题
4. 自动标签（地点/情绪/事件）
5. 自动形成故事线
6. 年末年度回忆
7. 基于日记的关系分析
8. 自然语言回忆搜索
